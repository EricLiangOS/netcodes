My Review of Github Copilot (I don't think it's going to take your job…yet)'

March 26th 2022

By Pedro Wayne

To start, I want to preface this with the fact that I am a student-developer, so take my opinion with a grain of salt. I have been using Github Copilot for around two weeks now, and here are my main findings.

What it does well

I am currently taking a couple of programming classes, and I decided to test GitHub Copilot on some of the assignments — it truly excelled. Yes, I had to check every solution for correctness, and sometimes I found mistakes. But most of the time, Copilot got it exactly right. I assume this is because many of these assignments have been written by others thousands of times. But don’t be mistaken, Copilot will not do these assignments for you. Every function has to be checked, and you will often find small mistakes. Moreover, the ethical implications of using Copilot are muddy.

Here is where I found it most useful (and more or less ethical)— I needed to repeat some lines of code a lot or quickly test my functions. For instance, we had to implement our version of sin and cos using recursion on another assignment. Here is what I told Copilot to do:

cout << “Testing actual sin and cos vs my sin and cos for values 0 to 1 in increments of 0.1\n”;

Here is what it generated

for (double i = 0; i <= 1; i += 0.1) {
    cout << "sin(" << i << ") = " << sin(i) << " vs " << s(i) << "\n";
    cout << "cos(" << i << ") = " << cos(i) << " vs " << c(i) << "\n";
}

Exactly what I needed.

It generally seems like the program is great at generating short simple algorithms and code snippets that don't require any difficult or over-arching tasks. Short and simple statements and words that Copilot and recognize, like sin and cos, are Copilot's cup of tea. However, the program definitely falters a bit when it comes to complicated tasks like algorithms.

What it does not do so well

When working on larger, more "real-life" projects, Github Copilot performs pretty poorly. Its suggestions are often, well…trash, and seem to be out of place. Unlike my assignments, when working on React projects or Node JS projects with an existing codebase, I just didn't find myself using Copilot often at all.

TLDR;

Copilot seems to work well for small, well-established tasks. However, in large codebases (where you will be spending most of your time as a developer probably), it's nothing more than above average code autocompletion. The time it takes for you to fill out Copilot is rarely worth the quick autocomplete, which you inevitably will have to go through and prune again.
