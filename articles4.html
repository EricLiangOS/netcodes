An Introduction to Graphs and the Depth First Search Algorithm

April 5th 2022

By Pedro Wayne

Trees and graphs are a core part of computer science because they allow us to visualize and store relationships between different objects. Of course, it’s also important to utilize these trees and data structures by finding out information about it, which is why there is a whole field of computer science related to graph algorithms. One of the most important graph algorithms is the depth first search (DFS).

Prerequisite Knowledge for DFS

As mentioned above, DFS is used on graphs and trees, which are data structures in computer science and come from mathematical graph theory. When you think of a graph, you probably think of the coordinates from math, where you can store coordinate points of different values that can represent different things. The points of these graphs also have distances between them that you can calculate, allowing you to plot street maps and other useful things.

The storing of values is quite important in computer science for algorithms like traffic, but all the unused space in the graph is very inefficient and wastes memory, so graphs are optimized to only keep track of the direct connections between locations. Trees are just a special type of graph that doesn't have any cycles.

The points of these graphs are normally called nodes or vertices, and these connections are called edges.

This brings up the question: How do you efficiently navigate across these graphs, whether for updating them or calculating a value?

This is where DFS comes in. The depth first search is one of the most optimized algorithms to traverse a graph, and it does so by continually going through unvisited edges and marking them as visited, until the algorithm terminates or has no more nodes to visit.

How DFS Works

I won’t be showing the actual code for the DFS since there’s a lot of different programming languages, but the basic idea is the same and can be implemented in any language once you understand it.

Here is the basic idea for the DFS:

  1. Visit a unvisited vertex s
  2. Mark s as visited
  3. Recursively visit each unvisited vertex attached to s

It looks easy right? That’s because the basic idea is incredibly easy to follow and you’ve probably done a similar algorithm when trying to brute force something. As you can see in the graph below, the algorithm tries to go further into the graph until it runs out of spaces to go; afterwards, it backtracks to the previous node and tries to go to other nodes. This process repeats until the entire graph is traversed.

The pseudocode is shown below for a more concrete implementation of the DFS algorithm:

dfs(node):

Through this algorithm, you can now visit all the nodes connected to the original node and do something with them, such as adding them up. One reason this works so seamlessly is because the recursive function call executes in the middle of the function, meaning you explore the next node first before going back.

Below is also a Python implementation:

# Using a Python dictionary to act as an adjacency list
graph = {
    'A' : ['B','C'],
    'B' : ['D', 'E'],
    'C' : ['F'],
    'D' : [],
    'E' : ['F'],
    'F' : []
}

visited = set() # Set to keep track of visited nodes.

def dfs(visited, graph, node):
    if node not in visited:
        print (node)
        visited.add(node)
        for neighbor in graph[node]:
            dfs(visited, graph, neighbor)

# Driver Code, prints all nodes
dfs(visited, graph, 'A')

Applications of DFS

Now that you know what the DFS is, what’s its purpose?

As mentioned above, one of the main uses of DFS is it allows for easy graph navigation to reach all the nodes, which is useful if you want to calculate something with all vertices, such as increasing its values (Although it’s possible to also do this without DFS).

Since the DFS goes through the graph with a distinct path each time, the DFS’s path also has a bunch of useful applications. For instance, DFS’ brute force approach is a good way to solve mazes since it will reach all the reachable cells, including the goal. Similarly, it can be used to solve even non-graph problems like Sudoku, with a bit of modification.

Finally, DFS has applications within computer science itself, acting as a building block for more difficult use cases. For instance, the algorithm is able to detect cycles or connected components within the graph because of its brute-force nature, and it can also be used for things such as topological sorting.

Ultimately, DFS is a simple yet highly useful algorithm that is incredibly prevalent in computer science. Through trial and error, DFS is able to solve many useful problems, and its modular nature allows for many modifications to solve even more problems. I highly recommend you try to implement it yourself in your preferred language to better understand the algorithm and its use cases.
